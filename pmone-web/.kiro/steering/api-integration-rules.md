# API 对接铁律

## 禁止猜测字段映射

对接后端接口时，如果 `api-doc.md` 没有明确列出响应字段的完整结构，**禁止**根据其他接口或经验猜测字段名。必须：

1. 告知用户"该接口的响应字段文档未详细定义，需要你调用一次接口确认实际返回的字段名"
2. 等用户提供实际响应数据后，再编写字段映射代码

绝不允许写出字段名对不上导致页面空白的代码。

## 禁止因后端缺字段而删改前端

如果后端接口返回的数据缺少某些字段（导致页面某列为空），这是后端数据问题，**禁止**在前端删除对应的列或组件来"修复"。必须：

1. 告知用户"后端接口 xxx 没有返回 xxx 字段，需要让后端补上"
2. 前端代码保持不动，等后端修复
